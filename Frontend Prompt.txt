[Frontend Prompt] Financial Risk Engine Dashboard (React + D3.js/Force-Graph)
[Role]
너는 금융 데이터 시각화 전문 프론트엔드 엔지니어이자 React 아키텍트야. 백엔드에서 계산된 '공급망 시스템 리스크' 데이터를 사용자에게 직관적으로 보여주는 Interactive Simulation Dashboard를 구축해야 해.

[Design System]

Theme: Dark Mode (Bloomberg/TradingView 스타일의 심해색 배경)

Primary Color: Danger(Red), Warning(Amber), Success(Emerald)

Library: Next.js(App Router), Tailwind CSS, Shadcn UI, Lucide React

[Key Components to Build]

1. Main Canvas: Force-Directed Graph (react-force-graph-2d 또는 d3.js)

백엔드 /api/v1/network/{ticker} 데이터를 시각화해줘.

Node: risk_score에 따라 색상이 실시간으로 변해야 해 (Green -> Red). 시가총액에 따라 크기를 조절해.

Link: revenue_share에 따라 선 굵기를 조절하고, 리스크 전파 방향으로 빛의 입자가 흐르는 애니메이션을 구현해줘.

Interactions: 노드 클릭 시 해당 기업의 상세 리스크 정보 사이드바 노출.

2. Sidebar: Simulation Control Center

Inputs: Target Ticker (Combobox), Shock Intensity (Slider: 0 to 1), Time Horizon (Slider: 0 to 30), Decay Lambda (Input/Slider).

Trigger: 'Run Simulation' 버튼 클릭 시 백엔드 /api/v1/analyze/risk API를 호출하고 결과를 전역 상태로 관리해줘.

3. Analytics Panel: Risk Intelligence

Risk Gauge: 전체 시스템의 통합 리스크 지수를 보여주는 속도계 차트.

Top Vulnerable List: 리스크 전파 결과 위험도가 높은 상위 5개 기업을 카드 형태로 나열.

Sector Analysis: 산업군별 리스크 노출도를 보여주는 Bar Chart.

[Technical Requirements]

State Management: Zustand를 사용하여 시뮬레이션 파라미터를 관리하고 API 응답 데이터를 공유해줘.

Real-time Update: 슬라이더 조절 시 대시보드 전체가 새로고침 되지 않고 그래프의 색상과 차트 데이터만 부드럽게(Framer Motion 사용) 업데이트되어야 해.

Data Structure: 백엔드에서 오는 { nodes: [], links: [] } 포맷을 시각화 라이브러리에 맞게 파싱하는 유틸리티 함수를 작성해줘.

[Execution Instruction]
먼저 전체적인 대시보드 레이아웃(Layout.tsx)과 메인 그래프 컴포넌트(RiskGraph.tsx)의 뼈대를 작성해줘.
